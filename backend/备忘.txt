备忘清单

- 重构判题核心：放弃“反射调用方法题”主路径，改为 javac 编译 + java 运行的真实 Java 执行链路
- 统一运行模型：首版只支持 Java 、单文件 Main.java 、标准输入输出题
- 拆分接口语义：区分“运行代码”和“提交评测”，不再共用一个判题入口
- 改造题目模型： code_problem 从 methodName/templateCode 模型升级为 OJ 题面模型
- 改造测试用例： code_test_case 从 input_json 改为纯文本 input_text ，按标准输入输出判题
- 扩展提交记录： code_submission 增加 problem_id 、源码、状态、编译输出、运行输出、测试点详情
- 新增运行接口：支持自定义 stdin ，返回 compileOutput/stdout/stderr/status/costMs
- 新增提交接口：使用后台测试用例判题，返回 submissionId/passCount/total/cases/status
- 新增提交记录接口：支持“我的提交列表”和“提交详情”
- 重写后端执行服务：使用 ProcessBuilder 创建独立 Java 子进程执行代码
- 保留基本保护：超时控制、输出长度限制、临时目录隔离、执行后清理文件
- 弱化旧方案： SecurityManager 和关键字黑名单只作过渡，不再作为核心机制
- 改造前端编程页：增加题面区、代码区、自定义输入区、编译输出区、运行输出区、评测结果区
- 按钮拆分：提供“运行代码”和“提交评测”两个操作
- 改造后台题目管理页：维护输入说明、输出说明、样例、时限、主类名、模板代码、文本测试用例
- 兼容旧数据：先加字段和新接口，旧的 run-problem 暂时保留，后续再下线
- 实施顺序：先打通真实编译运行，再做提交判题，再做后台管理和提交记录页
- MVP 目标：先做“可真实编译、可输入样例、可提交判题、可查看记录”的 Java 在线 OJ
建议优先级

- 第一优先：后端真实编译运行链路
- 第二优先：数据库结构升级
- 第三优先：前端 OJ 页面改造
- 第四优先：后台出题页改造
- 第五优先：稳定性与隔离增强
复杂度判断

- 低于 OJ：不涉及编译器、进程执行、判题链路、资源隔离
- 高于普通 CRUD：因为它同时包含用户中心、文件上传、权限控制、评论管理、用户自助修改，跨前后端多个模块
- 接近“完整社区化功能”的雏形：已经不是单点功能，而是账号体系扩展
这组功能包含的子模块

- 用户中心：个人资料、昵称、简介、头像、学习统计、我的答题/学习记录入口
- 个人主页：展示公开信息、学习概览、最近评论、最近学习动态
- 头像上传：前端上传组件、后端上传接口、文件存储、头像 URL 回显
- 评论区：课程评论、题目评论、评论列表、发表评论、编辑自己的评论、删除自己的评论
- 管理评论：管理员查看、删除、屏蔽评论，必要时支持按课程/题目/用户筛选
比“只加评论区”更复杂的原因

- 需要扩展 sys_user 相关资料字段
- 需要新增上传能力，不只是数据库增表
- 需要做“本人可改、管理员可管”的权限判断
- 评论对象不止一种，要同时支持课程和题目
- 页面不止一个，要新增用户中心页、个人主页页、评论区组件、管理端评论页
建议的数据结构

- sys_user 扩展字段：
  - avatar_url
  - bio
  - gender 或可选个人信息
  - last_login_time 可选
- 新增评论表 comment
  - id
  - user_id
  - target_type ： COURSE / QUESTION / CODE_PROBLEM
  - target_id
  - content
  - status ： NORMAL / HIDDEN / DELETED
  - create_time
  - update_time
- 如需图片头像本地存储：
  - 可先约定上传目录 + 数据库只存 URL
  - 首版不建议把图片二进制直接存数据库
建议的接口清单

- 用户中心
  - GET /api/user/profile
  - POST /api/user/profile/update
  - POST /api/user/avatar/upload
  - GET /api/user/{id}/homepage
- 评论区
  - GET /api/comment/list?targetType=&targetId=
  - POST /api/comment/add
  - POST /api/comment/update
  - DELETE /api/comment/{id}
- 管理员评论管理
  - GET /api/comment/admin/list
  - POST /api/comment/admin/hide
  - DELETE /api/comment/admin/{id}
权限规则建议

- 普通用户：
  - 可以发表自己的评论
  - 可以编辑自己的评论
  - 可以删除自己的评论
- 管理员：
  - 可以查看全部评论
  - 可以删除任意评论
  - 可以隐藏不当评论
- 未登录用户：
  - 可以查看评论
  - 不能发表评论和修改评论
前端页面建议

- 新增“用户中心页”
  - 头像、昵称、简介编辑
  - 我的学习进度
  - 我的评论
- 新增“个人主页页”
  - 用户公开资料
  - 最近评论
  - 最近学习记录摘要
- 在“课程详情页”增加评论区组件
- 在“题目详情页”增加评论区组件
- 在管理端增加“评论管理页”
推荐实施顺序

- 第一阶段：用户资料扩展 + 用户中心页
- 第二阶段：头像上传
- 第三阶段：评论表 + 评论接口
- 第四阶段：课程页和题目页挂接评论区
- 第五阶段：管理员评论管理页
